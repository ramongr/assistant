# RBS generator

How to run `assistant-rbs` against an `Assistant::Service` subclass and commit
the generated per-class signature so Steep can see the input readers produced
by `Service.input`. The matching site page is
[`docs/examples/rbs-generator.md`](../../docs/examples/rbs-generator.md), and
the rendered page is
<https://ramongr.github.io/assistant/#/examples/rbs-generator>.

Start with the [getting started guide](../../docs/getting-started.md) if this
is your first Assistant service.

## Files

| File | Role |
| --- | --- |
| [`create_user.rb`](./create_user.rb) | The service: required `email` / `name` inputs plus a nullable `role` that accepts `String` or `Symbol`. |
| [`sig/rbs_generator_example/create_user.rbs`](./sig/rbs_generator_example/create_user.rbs) | Generated output from `assistant-rbs`; commits the exact input reader types. |
| [`type_probe.rb`](./type_probe.rb) | Tiny Steep-checked call site that uses `#email`, `#name`, and `#role` as typed methods. |
| [`Steepfile`](./Steepfile) | Self-contained Steep target for this example. |

## Regenerating the signature

```sh
bundle exec exe/assistant-rbs examples/rbs_generator/create_user.rb \
  --output examples/rbs_generator/sig
```

The command rewrites only files whose first line is the generator marker:

```text
# Generated by assistant-rbs; do not edit.
```

Hand-written `.rbs` files without that marker are left untouched.

## Type-checking it

```sh
cd examples/rbs_generator
bundle exec steep check --steepfile Steepfile --jobs=1
```

The regression test
[`test/examples/rbs_generator_example_test.rb`](../../test/examples/rbs_generator_example_test.rb)
checks that the service runs and that regenerating the signature still matches
the committed file.
